Comment CBX fonctionne

Une expérience simple, structurée et sécurisée pour publier, trouver et conclure des opportunités d’affaires industrielles.

1️⃣

Inscription par profil

Chaque acteur choisit sa catégorie et complète un formulaire adapté à son statut : industrie, mine, fournisseur, intermédiaire, particulier, transporteur ou institution financière.

2️⃣

Validation et accès sécurisé

Le parcours prévoit un acte d’engagement, une signature numérique de démonstration et la génération d’un identifiant CBX avec code sécurisé.

3️⃣

Publication et mise en relation

Les commandes et offres sont publiées publiquement de façon anonyme. Les coordonnées complètes restent protégées jusqu’à validation d’accès.

Hébergement gratuit

La version actuelle de CBX est finalisée comme fichier index.html mono-fichier, public et trilingue, prêt à être déposé directement sur Netlify ou tout hébergement statique gratuit.

HTML mono-fichier Publication gratuite Aucun serveur requis

Mini mode d’emploi ultra simple (français)

  1. Téléchargez le fichier et renommez-le en index.html si nécessaire.
  2. Choisissez une plateforme gratuite : GitHub Pages, Netlify ou Cloudflare Pages.
  3. Importez le fichier ou le dossier du site : la plateforme vous donne ensuite un lien public à partager.

Le plus simple : Netlify ou Cloudflare Pages. Vous importez le fichier, vous récupérez le lien, et le site est en ligne.

Netlify Mise en ligne rapide Hébergement gratuit

Publication sur Netlify

Le livrable final de CBX peut être utilisé directement comme fichier index.html sur Netlify. Aucun build, aucun serveur et aucune configuration complexe ne sont nécessaires pour obtenir un lien public gratuit.

  1. Renommez le fichier en index.html si nécessaire.
  2. Ouvrez Netlify, puis choisissez l’option de dépôt simple du site.
  3. Glissez-déposez le fichier HTML ou le dossier du site dans l’interface.
  4. Netlify publie automatiquement le site et fournit un lien public gratuit.

Conseil : pour une mise en ligne ultra simple, Netlify est souvent la solution la plus rapide pour partager la maquette CBX.

GitHub Pages Domaine personnalisé gtf.CBS.app

Publication sur GitHub Pages + domaine personnalisé

Si vous préférez GitHub Pages, publiez le fichier index.html dans un dépôt public puis connectez le domaine gtf.CBS.app depuis les paramètres GitHub Pages et votre zone DNS.

  1. Créez un dépôt GitHub public et placez le fichier index.html à la racine.
  2. Ouvrez Settings > Pages, puis activez GitHub Pages depuis la branche principale.
  3. Dans GitHub Pages, renseignez le domaine personnalisé gtf.CBS.app.
  4. Dans le DNS du domaine, créez un CNAME pour gtf vers votre adresse GitHub Pages, puis attendez la propagation.

Important : si vous ne disposez pas de l’accès technique au dépôt GitHub ou au DNS du domaine, la connexion effective de gtf.CBS.app ne peut pas être exécutée ici ; seul ce mode d’emploi peut être fourni.
